# Design — RF-14.2: Comparação de Fotos de Progresso

**Data:** 2026-07-13
**Status:** Aprovado

## Contexto

O PRD exige comparação de fotos de progresso (RF-14.2). O backend já expõe `GET /api/progress/photos` retornando `Photo[]`. A view `PhotosView.vue` já exibe o grid de fotos com upload e delete. Nenhum mecanismo de comparação existe ainda.

---

## Escopo

Modificar apenas `frontend/src/views/student/PhotosView.vue`. Nenhum arquivo novo, nenhum novo store, nenhuma mudança no backend.

---

## Design

### Seleção no grid

Clicar em uma foto a seleciona (borda azul `ring-2 ring-blue-500` + ícone de checkmark branco no canto superior esquerdo). Clicar novamente a deseleciona. Máximo de 2 fotos selecionadas — clicar em uma 3ª foto enquanto já há 2 selecionadas não faz nada.

O botão de delete existente permanece, mas fica oculto quando a foto está selecionada (para evitar conflito visual com o checkmark).

### Barra de ações contextual

Exibida entre o cabeçalho e o grid, visível apenas quando há ≥ 1 foto selecionada:

| Estado | Conteúdo |
|--------|----------|
| 1 foto | "1 foto selecionada" + botão "Cancelar" |
| 2 fotos | "2 fotos selecionadas" + botão "Cancelar" + botão "Comparar" (primário) |

Botão "Cancelar" limpa `selectedPhotos`. Botão "Comparar" abre o dialog.

### Dialog de comparação

`style="width: min(860px, 95vw)"` para funcionar em mobile.

Layout interno — duas colunas com gap:

```
[ Antes — 10/03/2024 ]        [ Depois — 20/11/2024 ]
[ img h-64 object-cover ]     [ img h-64 object-cover ]
  "Notas: texto da foto"        "Notas: texto da foto"
```

- Ordenação: foto mais antiga à esquerda ("Antes"), mais recente à direita ("Depois"), independente da ordem de clique
- Cada coluna tem: label "Antes"/"Depois", data formatada, imagem com `object-cover`, notas (se existir)
- Footer do dialog: botão "Limpar seleção" (limpa `selectedPhotos` e fecha dialog) + botão "Fechar"
- Fechar o dialog via X ou "Fechar" **não** cancela a seleção — usuário pode querer comparar outras combinações

## Implementação

### Estado novo no script

```ts
const selectedPhotos = ref<Photo[]>([])  // máx 2

function toggleSelect(photo: Photo) {
  const idx = selectedPhotos.value.findIndex(p => p.id === photo.id)
  if (idx >= 0) {
    selectedPhotos.value.splice(idx, 1)
  } else if (selectedPhotos.value.length < 2) {
    selectedPhotos.value.push(photo)
  }
}

const comparePhotos = computed(() => {
  return [...selectedPhotos.value].sort(
    (a, b) => new Date(a.takenAt).getTime() - new Date(b.takenAt).getTime()
  )
})

const showCompare = ref(false)
```

### Alterações no template do grid

Cada item do grid passa a ter:
- `@click="toggleSelect(photo)"` no container (cursor pointer)
- `ring-2 ring-blue-500` condicionado a `selectedPhotos.includes`-by-id
- Checkmark (`pi pi-check-circle`) no canto superior esquerdo, visível apenas quando selecionado
- Botão delete: `v-show="!isSelected(photo)"` para sumir quando selecionado

### Imports adicionais

```ts
import { computed, ref, ... } from 'vue'  // computed já necessário
import Dialog from 'primevue/dialog'
import Button from 'primevue/button'       // já importado
```

`Dialog` é o único import novo de PrimeVue.

## Critérios de aceite

- Clicar em uma foto a seleciona com borda azul + checkmark
- Não é possível selecionar mais de 2 fotos
- Barra de ações aparece ao selecionar 1 ou 2 fotos
- Botão "Comparar" aparece somente com 2 fotos selecionadas
- Dialog exibe as duas fotos lado a lado: mais antiga à esquerda ("Antes"), mais recente à direita ("Depois")
- Cada foto mostra data e notas
- "Limpar seleção" fecha dialog e deseleciona tudo
- "Fechar" fecha dialog mas mantém seleção
- Delete de foto continua funcionando (quando a foto não está selecionada)
- Responsivo em mobile (`min(860px, 95vw)`)
